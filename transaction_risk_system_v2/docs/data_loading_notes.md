#  Загрузка и первичный анализ данных

Данный файл содержит заметки и типовые проблемы,
которые могут возникнуть на этапе загрузки и первичного осмотра данных
в рамках соревнований по анализу данных и машинному обучению.

---

##  1. Общий принцип работы с данными

- Все входные файлы (CSV, XLSX, JSON и др.) размещаются в папке `data/`
- Исходные файлы считаются **raw data** и не изменяются
- Работа ведётся с копиями данных в оперативной памяти (DataFrame)
- Запись на диск выполняется только при явном сохранении результата

---

##  2. Типы входных файлов

Обычно на соревнованиях встречаются:

- Основные таблицы с событиями (transactions, logs и т.п.)
- Таблицы с целевой переменной (labels, train, target)
- Справочники (коды, категории, типы операций)
- Метаданные и описательные таблицы

Не все файлы используются напрямую для обучения модели.

---

##  3. Типовые проблемы при загрузке файлов

### 3.1 Нестандартный разделитель

Признак:
- ошибка `Error tokenizing data`

Причина:
- разделитель `;`, `\t`, `|` вместо `,`

Решение:
```python
pd.read_csv(path, sep=";")
pd.read_csv(path, sep="\t")
pd.read_csv(path, engine="python")
